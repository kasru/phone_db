Задание 1:

1 случай:
Один раз в сутки поступают данные в виде набора файлов формата CSV(текст с разделителями). 
Новые данные полностью заменяют собой старые в памяти.
Структура записи(строки) в csv файле:
 <номер телефона в международном формате (до 19 знаков, возможны символы 0-9, A-F)>
<текстовый комментарий (может отсутствовать)>
<дата окончания действия (может отсутствовать)> 
Объём поступающих данных - до 300 миллионов записей(т.е. строк в CSV файлах).

2 случай:
Далее в течение суток поступают файлы инкрементов (т.е. инструкций для изменения данных загруженных ранее), также в CSV формате. Эти файлы содержат записи для удаления, добавления, модификации данных по телефонному номеру. 
Скорость поступающих данных - приблизительно до 10 записей в секунду. 
Структура записи:
<тип операции A|D|U (т.е. add, delete или update>
<номер телефона в международном формате (до 19 знаков, возможны символы 0-9, A-F)>
<текстовый комментарий (может отсутствовать)>
<дата окончания действия (может отсутствовать)> 

Требуется: 
реализовать загрузку поступающих данных в память(в обеих случаях) и подготовку их для дальнейшего использования таким образом, чтобы:
1) Загрузка входного файла в 1-ом случае работало бы максимально быстро (менее минуты для объема порядка 300 миллионов записей)
2) Занимаемая данными память была бы как можно меньшей.
3) В загруженных данных можно было быстро найти запись с заданным номером телефона.
4) Данные можно было сохранить в файл на диске в порядке возрастания номеров.
5) Загрузка новых данных(в обоих случаях) не приводила бы к существенным задержкам при использовании предыдущего варианта данных.
6) Язык программирования - С. Разделитель - ";", между записями - "\n"
